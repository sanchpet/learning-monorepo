# learning-monorepo

Единый дом для учебного кода — упражнений, code-along и разминок со всех курсов
и материалов. Не свалка: чистые коммиты, изолированные окружения, навигация,
которая не врёт.

Здесь видно **как** я учусь: подход, регулярность, аккуратность. Доведённые до
продукта вещи живут в отдельных репозиториях (см. ниже) — там видно **что** я
умею довести до конца.

## Философия

Учебный код неоднороден. В нём два разных по природе типа артефактов:

- **Разминка** — code-along по урокам, выбрасываемый код. Цель — понять
  механизм, а не показать кому-то. Живёт здесь, в монорепо.
- **Витрина** — то, что развивается дальше как инструмент или показывается
  работодателем. Выносится в отдельный репозиторий с чистым README и
  презентабельной историей.

Граница между ними проявляется не на старте, а задним числом. Поэтому правило —
**по умолчанию всё сюда, выносим по факту**, когда сработал конкретный повод.

## Критерии выделения в отдельный репозиторий

Проект уходит из монорепо в свой репозиторий, если выполнен хотя бы один пункт:

1. **Показ** — хочу показать конкретным людям как самостоятельную работу.
2. **Деплой** — разворачиваю в личную инфраструктуру.
3. **Развитие** — продолжаю и развиваю как инструмент.
4. **Курс** — этого требует условие учебного курса (например, capstone).

До срабатывания одного из поводов код остаётся здесь. Решение одностороннее:
вынести легко, вернуть обратно — незачем.

## Лейтмотив

При создании любого учебного кода задаю вопрос:

> **Как применить это знание с пользой для моей инфраструктуры и развить
> в практически применимое?**

Это **линза, чтобы заметить возможность**, а не мандат. Ответ «нет, это чистая
разминка, выбрасываю» — полностью легитимный. Фиксирую только когда ответ «да,
тут есть зерно» → проект становится кандидатом на выделение по критериям выше.

## Конвенции

- **Изолированное окружение на проект.** У каждого курса (и при необходимости
  упражнения) — собственный `mise.toml`, пиннящий стек. Никакого общего `venv`
  на весь репо: зависимости курсов конфликтуют.
- **Стек добавляется лениво.** Появился курс на новом языке — тогда и заводим
  его окружение. Не раскладываем полиглот-леса заранее.
- **Один курс — один каталог** в `courses/`. Внутри — упражнения/модули.
- **Чистые коммиты.** Атомарные, осмысленные сообщения — репо презентует.

## Как это устроено

Навигация ниже (между маркерами `AUTO:INDEX`) **генерируется автоматически** из
дерева `courses/` скриптом `scripts/gen_index.py`, повешенным на git
`pre-commit` hook. Дерево каталогов — источник истины, README лишь отражает его.
Руками индекс не правим — он перезапишется при следующем коммите.

Активировать hook после клонирования:

```bash
git config core.hooksPath .githooks
```
